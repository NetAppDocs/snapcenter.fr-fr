= Configurer les identifiants pour le cluster PostgreSQL
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Vous devez configurer les identifiants de l'utilisateur du cluster PostgreSQL pour effectuer des opérations de protection des données.

.Avant de commencer
* Si le cluster PostgreSQL ne dispose pas d'identifiants valides, un cadenas rouge s'affiche uniquement pour les ressources détectées automatiquement. Si, lors d'une détection ultérieure, les identifiants configurés s'avèrent incorrects ou ne permettent pas d'accéder au cluster, le cadenas rouge réapparaît.
* Vous devez configurer des identifiants de cluster PostgreSQL valides pour pouvoir protéger la ressource ou l'ajouter à un groupe de ressources afin d'effectuer des opérations de protection des données.
* L'utilisateur PostgreSQL configuré dans les identifiants doit disposer des privilèges suffisants pour accéder au cluster et effectuer des opérations de sauvegarde et de restauration. Il doit au minimum posséder le rôle *SUPERUSER* ou des privilèges équivalents (tels que les permissions *pg_read_all_data*, *pg_write_all_data* et *replication*).
* SnapCenter utilise les identifiants configurés pour se connecter à l'instance PostgreSQL via le client *psql* ou *libpq*. Assurez-vous que les outils clients PostgreSQL sont installés sur l'hôte du plug-in.



NOTE: Avant de configurer le cluster, vous devez d'abord ajouter les identifiants PostgreSQL à SnapCenter.

.Étapes
. Dans le volet de navigation de gauche, sélectionnez *Ressources* puis sélectionnez SnapCenter Plug-in for PostgreSQL cluster dans la liste.
. Dans la page Ressources, sélectionnez le type de ressource dans la liste *Affichage*.
. (Facultatif) Sélectionnez image:../media/filter_icon.gif["icône de filtre"] et sélectionnez le nom de l'hôte.
+
Vous pouvez ensuite sélectionner image:../media/filter_icon.gif["icône de filtre"] pour fermer le volet de filtre.

. Sélectionnez le cluster, puis sélectionnez *Configurer les identifiants*.
. Sélectionnez l’identifiant qui a été configuré dans SnapCenter pour l’utilisateur du cluster PostgreSQL. L’identifiant doit inclure un nom d’utilisateur et un mot de passe valides, avec des privilèges suffisants pour effectuer des opérations de protection des données.
. Sélectionnez *OK*.


Vous pouvez modifier la configuration du cluster depuis la page Topologie.
